## 2026-05-14 - explicit focus styles using design tokens
**Learning:** For apps using plain CSS without a global reset and no framework utility classes like focus-visible, it's crucial to explicitly bind `:focus-visible` pseudo-class rules directly to existing interactive component classes (e.g. `.primary-button:focus-visible`, `.nav-stack a:focus-visible`) to maintain accessibility. Relying on default browser outlines can cause inconsistency or invisibility depending on background colors.
**Action:** When working with vanilla CSS, append explicit `:focus-visible` selectors to existing class declarations (e.g., `.nav-stack a:focus-visible, .primary-button:focus-visible`) and use the main `var(--accent)` design token for the `outline` with an appropriate `outline-offset`.
