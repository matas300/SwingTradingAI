## 2025-04-08 - Global Missing Focus Indicators
**Learning:** The entire application lacked visible focus states (`:focus-visible`) for core interactive elements (links, buttons, inputs, selects, textareas). Keyboard users would have no clear indication of their current active element, rendering keyboard navigation extremely difficult. This pattern suggests that visual styles were primarily designed for pointer devices without explicit consideration for keyboard accessibility.
**Action:** Always verify if a reset or base CSS setup strips default outlines, and explicitly define `:focus-visible` states using theme variables (like `--accent`) globally for all interactive tag types (`a`, `button`, `input`, `select`, `textarea`) before assuming they are accessible by default.
